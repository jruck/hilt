# X long-post extraction

This generation is a narrow fix for X/Twitter bookmarks where the bookmark-list response only returned the truncated public `text`, even though the post had a full `note_tweet`.

**Check these specifically:**

- Long X posts should carry the full source body in Raw Content, not just the first 280-ish characters.
- Numbered/listed posts should preserve the actual numbered claims in the digest or takeaways. They should not summarize only item 1.
- Thread/list roots that are still partial should be labeled as source-limited or warm, not silently treated as hot.
- Redigested saved references should keep the normal free-form Library body shape, not regress to a forced Summary/Key Points template when the reweave generated better sections.

The motivating failure was the Ahrefs AI-search bookmark (`2061796432534003866`): the source had 10 numbered findings, but the saved note captured only finding 1.
