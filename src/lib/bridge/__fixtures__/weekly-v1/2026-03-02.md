	───

	Hilt Feature Request: Task List Subheading Support

	What: Weekly task lists now use ### Subheading (h3) within the file to group tasks into themed sections (Personal, Writing, Product, etc.). Hilt needs to recognize these as visual group separators.

	Current behavior: Hilt expects ## Tasks as the single heading containing all tasks. When a user drags a task to reorder it, Hilt rewrites the file with ## Tasks and a flat task list, stripping all ### subheadings. All content is preserved but the grouping is lost.

	Desired behavior:

	1\. Parse ### headings within the task section as group labels

	2\. Render them as visual separators/section headers in the task list UI

	3\. When rewriting the file (drag reorder, checkbox toggle, etc.), preserve the ### headings and keep tasks within their respective groups

	4\. Drag reorder should work within a group. Dragging between groups = moving the task under a different ### heading.

	File structure example:

	\# Week of 2026-03-02

	\### Personal

	\- \[ \] Collect tax info

	\- \[x\] Set up hyper key

	\### Product

	\- \[ \] Auto-reply scope and design

	\- \[ \] PulseM Insights pilot

	\### Agent Infrastructure

	\- \[ \] Memory & Search

	\- \[ \] Reference Library

	\### Done

	\- \[x\] Jason access call

	Why: Task lists are getting long (15-20 items). Flat lists are overwhelming. Subheadings let Justin group related work visually without changing the underlying task structure. Agents also use these groups when reviewing/updating the list.

	Edge cases to consider:

	• Tasks not under any ### heading (backward compat — treat as ungrouped)

	• ### Done section at the bottom for completed tasks

	• The ## Tasks heading may or may not be present — some files use it, the new format just uses ### directly under # Week of...

	───
