# Embedded Video Recovery

Thin saved pages now get one source-quality fallback before digestion: when an explicit study save
has little readable prose but exposes a primary embedded video, Hilt retrieves captions or, only when
needed, downloads audio for transcription. The transcript becomes Raw Content and feeds the normal
digest, connection, and evaluation path.

The motivating case is **Notion Ship OS**. Its page exposed only a tagline and a Play control, while
the substantive 63-second walkthrough lived in a serialized Vimeo player. Review that the regenerated
note reflects the actual triage → planning → build → review → ship workflow, embeds the Vimeo player,
stores transcript provenance, and no longer describes the source as content-free.

Cost/quality gates are part of the iteration: explicit saves only by default, thin prose only, videos
under 20 seconds skipped, captions before audio transcription, at most three candidates, and failed
required transcripts use the normal two-attempt capture policy instead of producing a confident stub.
