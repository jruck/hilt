# AI Operating Loop

## Media

![AI operating loop](/demo/library-ai-loop.svg)

## Summary

The piece describes a practical loop for AI-assisted work: capture weak signals, digest them into shared context, connect them to active projects, and resurface the small set that deserves attention.

## Key Points

- Durable knowledge starts with consistent capture, not perfect organization.
- Summaries become useful when they link back to active projects and decisions.
- Review queues should preserve "maybe" items without polluting permanent notes.
- The system should make source health visible so silent ingestion failures are caught early.

An AI operating loop has four jobs. First, it captures source material from places where the user already expresses intent. Second, it digests each item into a useful summary, key points, and possible connections. Third, it separates durable saves from temporary candidates so the permanent archive stays useful. Fourth, it resurfaces a small number of timely items in a feed or briefing. The loop is strongest when every step leaves enough evidence to debug failures later.
